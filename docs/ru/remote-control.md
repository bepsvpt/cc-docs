# Продолжайте локальные сеансы с любого устройства с помощью Remote Control

> Продолжайте локальный сеанс Claude Code со своего телефона, планшета или любого браузера, используя Remote Control. Работает с claude.ai/code и мобильным приложением Claude.

<Note>
  Remote Control доступен на всех планах. Администраторы Team и Enterprise должны сначала включить Claude Code в [параметрах администратора](https://claude.ai/admin-settings/claude-code).
</Note>

Remote Control подключает [claude.ai/code](https://claude.ai/code) или приложение Claude для [iOS](https://apps.apple.com/us/app/claude-by-anthropic/id6473753684) и [Android](https://play.google.com/store/apps/details?id=com.anthropic.claude) к сеансу Claude Code, работающему на вашем компьютере. Начните задачу за своим столом, а затем продолжите её со своего телефона на диване или в браузере на другом компьютере.

Когда вы запускаете сеанс Remote Control на своем компьютере, Claude продолжает работать локально всё время, поэтому ничего не переходит в облако. С помощью Remote Control вы можете:

* **Использовать полную локальную среду удалённо**: ваша файловая система, [MCP servers](/ru/mcp), инструменты и конфигурация проекта остаются доступными
* **Работать с обеих поверхностей одновременно**: беседа остаётся синхронизированной на всех подключённых устройствах, поэтому вы можете отправлять сообщения из терминала, браузера и телефона взаимозаменяемо
* **Пережить перебои**: если ваш ноутбук переходит в режим сна или сетевое соединение прерывается, сеанс автоматически переподключается, когда ваш компьютер снова подключится

В отличие от [Claude Code в веб-версии](/ru/claude-code-on-the-web), который работает на облачной инфраструктуре, сеансы Remote Control работают непосредственно на вашем компьютере и взаимодействуют с вашей локальной файловой системой. Веб-интерфейс и мобильные интерфейсы — это просто окно в этот локальный сеанс.

<Note>
  Remote Control требует Claude Code версии 2.1.51 или позже. Проверьте вашу версию с помощью `claude --version`.
</Note>

На этой странице рассматривается настройка, способы запуска и подключения к сеансам, а также сравнение Remote Control с Claude Code в веб-версии.

## Требования

Перед использованием Remote Control убедитесь, что ваша среда соответствует этим условиям:

* **Подписка**: доступна на планах Pro, Max, Team и Enterprise. Администраторы Team и Enterprise должны сначала включить Claude Code в [параметрах администратора](https://claude.ai/admin-settings/claude-code). API ключи не поддерживаются.
* **Аутентификация**: запустите `claude` и используйте `/login` для входа через claude.ai, если вы ещё этого не сделали.
* **Доверие рабочей области**: запустите `claude` в каталоге вашего проекта хотя бы один раз, чтобы принять диалог доверия рабочей области.

## Запуск сеанса Remote Control

Вы можете запустить новый сеанс непосредственно в Remote Control или подключиться к уже работающему сеансу.

<Tabs>
  <Tab title="Новый сеанс">
    Перейдите в каталог вашего проекта и запустите:

    ```bash  theme={null}
    claude remote-control
    ```

    Процесс продолжает работать в вашем терминале, ожидая удалённых подключений. Он отображает URL сеанса, который вы можете использовать для [подключения с другого устройства](#connect-from-another-device), и вы можете нажать пробел, чтобы показать QR-код для быстрого доступа со своего телефона. Пока активен удалённый сеанс, терминал показывает статус подключения и активность инструментов.

    Эта команда поддерживает следующие флаги:

    * **`--name "My Project"`**: установите пользовательское название сеанса, видимое в списке сеансов на claude.ai/code. Вы также можете передать имя как позиционный аргумент: `claude remote-control "My Project"`
    * **`--verbose`**: показать подробные журналы подключения и сеанса
    * **`--sandbox`** / **`--no-sandbox`**: включить или отключить [sandboxing](/ru/sandboxing) для изоляции файловой системы и сети во время сеанса. Sandboxing отключен по умолчанию.
  </Tab>

  <Tab title="Из существующего сеанса">
    Если вы уже находитесь в сеансе Claude Code и хотите продолжить его удалённо, используйте команду `/remote-control` (или `/rc`):

    ```text  theme={null}
    /remote-control
    ```

    Передайте имя как аргумент, чтобы установить пользовательское название сеанса:

    ```text  theme={null}
    /remote-control My Project
    ```

    Это запускает сеанс Remote Control, который переносит историю вашей текущей беседы и отображает URL сеанса и QR-код, которые вы можете использовать для [подключения с другого устройства](#connect-from-another-device). Флаги `--verbose`, `--sandbox` и `--no-sandbox` недоступны с этой командой.
  </Tab>
</Tabs>

### Подключение с другого устройства

После активации сеанса Remote Control у вас есть несколько способов подключиться с другого устройства:

* **Откройте URL сеанса** в любом браузере, чтобы перейти непосредственно к сеансу на [claude.ai/code](https://claude.ai/code). Как `claude remote-control`, так и `/remote-control` отображают этот URL в терминале.
* **Отсканируйте QR-код**, показанный рядом с URL сеанса, чтобы открыть его непосредственно в приложении Claude. С помощью `claude remote-control` нажмите пробел, чтобы переключить отображение QR-кода.
* **Откройте [claude.ai/code](https://claude.ai/code) или приложение Claude** и найдите сеанс по имени в списке сеансов. Сеансы Remote Control показывают значок компьютера с зелёной точкой статуса, когда они в сети.

Удалённый сеанс получает своё имя из аргумента `--name` (или имени, переданного в `/remote-control`), вашего последнего сообщения, вашего значения `/rename` или "Remote Control session", если нет истории беседы. Если в среде уже есть активный сеанс, вас спросят, продолжить ли его или начать новый.

Если у вас ещё нет приложения Claude, используйте команду `/mobile` внутри Claude Code, чтобы отобразить QR-код загрузки для [iOS](https://apps.apple.com/us/app/claude-by-anthropic/id6473753684) или [Android](https://play.google.com/store/apps/details?id=com.anthropic.claude).

### Включение Remote Control для всех сеансов

По умолчанию Remote Control активируется только при явном запуске `claude remote-control` или `/remote-control`. Чтобы включить его автоматически для каждого сеанса, запустите `/config` внутри Claude Code и установите **Enable Remote Control for all sessions** на `true`. Установите его обратно на `false`, чтобы отключить.

Каждый экземпляр Claude Code поддерживает один удалённый сеанс одновременно. Если вы запускаете несколько экземпляров, каждый получает свою собственную среду и сеанс.

## Подключение и безопасность

Ваш локальный сеанс Claude Code выполняет только исходящие HTTPS запросы и никогда не открывает входящие порты на вашем компьютере. Когда вы запускаете Remote Control, он регистрируется в API Anthropic и опрашивает работу. Когда вы подключаетесь с другого устройства, сервер маршрутизирует сообщения между веб-клиентом или мобильным клиентом и вашим локальным сеансом через потоковое соединение.

Весь трафик проходит через API Anthropic через TLS, тот же транспортный протокол безопасности, что и любой сеанс Claude Code. Соединение использует несколько краткосрочных учётных данных, каждые из которых ограничены одной целью и истекают независимо.

## Remote Control в сравнении с Claude Code в веб-версии

Remote Control и [Claude Code в веб-версии](/ru/claude-code-on-the-web) оба используют интерфейс claude.ai/code. Ключевое различие заключается в том, где работает сеанс: Remote Control выполняется на вашем компьютере, поэтому ваши локальные MCP servers, инструменты и конфигурация проекта остаются доступными. Claude Code в веб-версии выполняется в управляемой Anthropic облачной инфраструктуре.

Используйте Remote Control, когда вы находитесь в процессе локальной работы и хотите продолжить с другого устройства. Используйте Claude Code в веб-версии, когда вы хотите запустить задачу без какой-либо локальной настройки, работать с репозиторием, который у вас не клонирован, или запускать несколько задач параллельно.

## Ограничения

* **Один удалённый сеанс одновременно**: каждый сеанс Claude Code поддерживает одно удалённое подключение.
* **Терминал должен оставаться открытым**: Remote Control работает как локальный процесс. Если вы закроете терминал или остановите процесс `claude`, сеанс завершится. Запустите `claude remote-control` снова, чтобы начать новый сеанс.
* **Продолжительный сбой сети**: если ваш компьютер включен, но не может подключиться к сети более чем примерно на 10 минут, сеанс истекает и процесс завершается. Запустите `claude remote-control` снова, чтобы начать новый сеанс.
